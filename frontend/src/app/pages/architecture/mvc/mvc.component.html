La arquitectura Modelo-Vista-Controlador (MVC) es un patrón de diseño de software que divide una aplicación en tres partes interconectadas: el Modelo (datos y lógica de negocio), la Vista (interfaz de usuario) y el Controlador (gestiona la interacción del usuario y actualiza el Modelo y la Vista). MVC promueve la separación de responsabilidades, facilita el desarrollo, la mantenibilidad y la escalabilidad. El Controlador recibe la entrada del usuario, interactúa con el Modelo para obtener o modificar datos, y luego actualiza la Vista para reflejar los cambios. Esta separación permite que múltiples Vistas utilicen el mismo Modelo, y facilita las pruebas unitarias.
